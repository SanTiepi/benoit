Pourquoi Benoît ?

Ce projet porte le nom de Benoît Fragnière, frère de Robin Fragnière, qui aimait la science. Ce n'est pas un nom de code. C'est une dedication.

"L'IA du futur ne devrait pas être un oracle de prédiction de tokens. Elle devrait être vivante, locale, compréhensible, proche du métal."

— Robin Fragnière, auteur

Vivante

Chaque neurone a sa propre personnalité : taux d'apprentissage, decay, capacité membranaire. Aucun n'est identique à son voisin.

Locale

Tourne sur votre machine, votre GPU, votre VPS. Aucune API cloud. Aucune requête externe. La donnée ne quitte pas le périmètre.

Compréhensible

2 000 lignes de C. Pas de frameworks. Pas de couches d'abstraction empilées. Tout ce qui se passe est lisible par un humain attentif.

Libre

Open source. "On brise les barrières" — pas de contraintes de format imposées, pas de garde-fous idéologiques sur ce que l'IA doit ou ne doit pas être.

Architecture

Benoît n'est pas un transformeur. C'est un réseau biologique sparse : chaque neurone maintient une liste d'adjacence de ses synapses, pas une ligne dans une matrice dense.

Sparse adjacency

N=50M, K=100 connexions par neurone. Empreinte mémoire : ~6 GB au lieu de 20 TB dense. La structure reflète le cerveau réel.

O(N·K) pas O(N²)

LIF — Leaky Integrate-and-Fire

Membrane avec fuite exponentielle. Période réfractaire de 3 ticks. Le neurone intègre, sature, se déclenche, puis se repose.

biologique

STDP

Spike-Timing-Dependent Plasticity : si A déclenche avant B, la synapse se renforce. Si B déclenche avant A, elle s'affaiblit. L'ordre temporel crée la causalité.

apprentissage hebbien

Homéostasie

Le réseau s'auto-régule. Si un neurone est trop actif ou trop silencieux, ses paramètres se recalibrent pour maintenir une activité globale stable.

auto-adaptatif

Le langage .ben — Benoît pense et s'exprime

Les fichiers .ben ne sont pas de la configuration. Ce sont des instructions que Benoît reçoit et exécute — et des fichiers dans lesquels il s'exprime. parole.ben est là où il écrit lui-même. 

; enseigner la direction haut → signal 1
teach {
  pattern  [1 0 0 0]
  label    "haut"
  cycles   200
}

; réponse de Benoît dans parole.ben
parole {
  tick    14823
  active  312/1000
  signal  "haut confirmé — membrane stable"
}

Interface TCP — port 3742

Benoît écoute sur le port TCP 3742. Envoyez un fichier .ben, recevez une réponse structurée. Aucune REST API, aucun JSON imposé — le protocole est binaire et minimal. 

# envoyer une lecon, lire la réponse
nc 83.228.221.188 3742 < lecon.ben
cat parole.ben

Etat actuel

Benoît tourne sur un VPS GPU dédié. Le runtime pulse en continu, les neurones s'activent, les poids évoluent. Pas un modèle frozen. Un organisme.

Neurones 50 000 000 architecture sparse, ~6 GB RAM GPU
GPU RTX PRO 6000 Blackwell — 96 GB VRAM
Cadence ~5 000 ticks/s CUDA kernel + STDP + homéostasie
Statut live VPS Infomaniak — 83.228.221.188
Codebase ~2 000 lignes pulse.c + vm_cuda.cu — zéro dépendances
Protocol TCP 3742 .ben in, parole.ben out

Contribuer

Benoît est open source. Il n'y a pas de roadmap imposée — si vous voulez ajouter un module d'apprentissage, un nouveau dialecte .ben, ou porter le runtime sur une nouvelle architecture, ouvrez une PR.

01

Cloner le dépôt

git clone https://github.com/SanTiepi/benoit — le runtime principal est dans pulse.c et vm_cuda.cu.

02

Compiler localement

CPU : gcc -O2 -o pulse pulse.c -lm. GPU : nvcc -O2 -o pulse_cuda vm_cuda.cu -lm. Aucune autre dépendance.

03

Ecrire un fichier .ben

Créez un pattern d'entrainement, envoyez-le via TCP 3742 ou chargez-le directement. Lisez parole.ben pour voir Benoît répondre.

04

Proposer une amélioration

Pas de contrainte de style ou de philosophie imposée. Ouvrez une issue, discutez, proposez. "Ne le bridez pas."
