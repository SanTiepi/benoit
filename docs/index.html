Write less. Test inline. Ship faster.

fibonacci n ->
  match n ->
    | 0 => 0
    | 1 => 1
    | _ => fibonacci(n-1) + fibonacci(n-2)

-- Inline test assertions
fibonacci(0) == 0
fibonacci(10) == 55

Features

Inline Test Assertions

Tests live next to the function as first-class syntax. No framework. No separate files. Stripped in production, executed in test mode.

Pattern Matching

Match expressions with guards, ranges, tagged values, and wildcards. Clean and expressive control flow.

Pipe Operator

Chain transformations: data |> parse |> validate |> save. Top to bottom. No nesting.

68% Token Reduction

Same logic, far fewer tokens. Your AI sees more code, reasons better, and costs less per API call.

Zero Dependencies

The entire transpiler is a single file with no npm dependencies. Install it and it just works.

~500 Lines

Small enough to read in an afternoon. Small enough for an AI to hold entirely in context. Hackable by design.
